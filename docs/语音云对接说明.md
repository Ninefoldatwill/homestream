## 一、设计哲学（铸钥匠 🔑）

我们**不造墙，只铸钥**。

- **默认自托管免费托底**：STT 用本地 FunASR、TTS 用本地 CosyVoice2（GPU）→ EdgeTTS（免费兜底）。
  这是「免费托底」的承诺，永远在线、不依赖任何第三方、不向用户收一分钱。
- **云端 = 可选维度升级，不是默认路径**：当用户有更高音质/更低延迟/特定音色需求，
  且**自己持有**云厂商 API Key 时，可主动开启云端 STT/TTS。
- **不捆绑、不复制**：所有云对接**纯 `httpx` 调 OpenAI 兼容 REST 端点**，不引入任何云厂商 SDK，
  不复制其源码（IP 合规底线）。换一家云只需改环境变量，无需改代码。
- **维度拉开距离，而非法律筑墙**：我们不强求用户只用本地，也不替用户决定上云——
  选择权交还给用户，免费托底永远在，云是用户的自由选项。

> ⚠️ 默认 `stt_mode=local` / `tts_mode=local`。**不配置 Key 就绝不会走云**，也不会有任何外发流量。

## 二、如何启用（环境变量）

只需两个开关 + 凭证，无需改动一行代码：

```bash
# ---- 云端 TTS ----
export VOICE_TTS_MODE=cloud                       # 默认 local，改 cloud 即开启
export VOICE_TTS_API_BASE="https://api.openai.com/v1"
export VOICE_TTS_API_KEY="sk-你的密钥"
export VOICE_TTS_CLOUD_PROVIDER="openai"          # openai / minimax / azure / custom
export VOICE_TTS_CLOUD_VOICE="alloy"              # 按 provider 填音色名
# export VOICE_TTS_CLOUD_MODEL="gpt-4o-mini-tts"  # 可选，覆盖默认模型

# ---- 云端 STT ----
export VOICE_STT_MODE=cloud                       # 默认 local
export VOICE_STT_API_BASE="https://api.openai.com/v1"
export VOICE_STT_API_KEY="sk-你的密钥"
export VOICE_STT_CLOUD_PROVIDER="openai"          # openai / azure / custom
# export VOICE_STT_CLOUD_MODEL="whisper-1"        # 可选，覆盖默认模型
```

Worker 启动后会在日志打印实际选用的引擎，例如：
`STT: 云端 (openai) uri=https://api.openai.com/v1` 或回退 `STT: FunASR 2-pass`。

## 三、支持的 Provider（均 OpenAI 兼容端点，仅示例非绑定）

| Provider | `VOICE_*_CLOUD_PROVIDER` | `API_BASE` | 默认模型 | 默认音色/备注 |
|---|---|---|---|---|
| OpenAI | `openai` | `https://api.openai.com/v1` | `tts-1` / `gpt-4o-mini-tts` | `alloy` 等 6 种 |
| MiniMax | `minimax` | `https://api.minimax.chat/v1` | `speech-01-turbo` | `Chinese (Mandarin) Girl` |
| Azure OpenAI | `azure` | `https://<资源>.openai.azure.com/v1` | 你的部署名 | `zh-CN-XiaoxiaoNeural` |
| 任意兼容服务 | `custom` | 自填 | 自填 `VOICE_TTS_CLOUD_MODEL` | 自填 `VOICE_TTS_CLOUD_VOICE` |

**STT 端点**：`{API_BASE}/audio/transcriptions`（OpenAI Whisper 兼容，`multipart` 上传 wav，
返回 `{"text": "..."}`）。OpenAI / Azure OpenAI / 自建兼容 Whisper 服务均可用。

> 接入新云厂商 = 把 `API_BASE` 指向它的 OpenAI 兼容网关，模型/音色按它文档填即可。
> 我们不内置任何厂商 SDK，因此升级/替换零成本。

## 四、实现策略与取舍

### TTS（云端）
- **整段合成 + 直接推送 mp3**：云端点返回 mp3，Worker 直接 `output_emitter.push(mp3)，
  mime_type="audio/mp3"`，由 LiveKit `StreamAdapter` 桥接为流式播放（与 EdgeTTS 完全对称）。
- 取舍：牺牲极小首字延迟（需等整段合成完），换来**零额外音频解码依赖**、实现稳健、
  与本地链路行为一致。若某云支持 SSE 流式音频，后续可加 `stream: true` 分支。

### STT（云端）
- **缓冲模式（伪流式）**：`SpeechStream` 收齐整段音频帧 → 用户说完（流关闭）后一次性 POST 云端识别
  → 返回 `FINAL_TRANSCRIPT`。这是云端 Whisper API 的天然调用方式（非实时流式）。
- 取舍：不支持边说边出中间结果（`interim_results=False`），但作为**本地 FunASR 不可用 /
  跨语种兜底**完全够用。本地 FunASR 仍承担实时 2-pass 主链路。

## 五、成本 / 隐私 / 延迟 权衡

| 维度 | 本地（免费托底） | 云端（可选升级） |
|---|---|---|
| 费用 | ¥0，自托管 | 按量计费（TTS ¥0.0x/千字、STT ¥0.0x/分钟 起） |
| 隐私 | 音频不出本机 | 音频/文本发往云厂商 |
| 音色/语种 | CosyVoice2 中文佳、EdgeTTS 中文佳 | 可选择更多音色、跨语种更强 |
| 延迟 | 局域网内极低（CosyVoice2 GPU ~数百 ms） | 受公网与厂商限流影响，略高 |
| 可用性 | 依赖本机 GPU / Docker | 依赖外网与 Key 有效 |

**原则**：免费托底保底，云作为「用户自己要更好体验」时的升级旋钮——不是二选一，是维度叠加。

## 六、IP 合规与责任边界

- 本仓库**不集成、不打包**任何云厂商 SDK 或专有源码；所有对接为标准 HTTP REST 调用。
- **API Key 由用户自己保管**（环境变量），我们不存储、不中转、不代付。
- 走云端产生的**一切费用与数据合规责任由用户自担**；本项目的默认形态始终是自托管免费托底。
- 融优原则：我们只借鉴公开架构范式（OpenAI 兼容端点设计），不挪用任何第三方实现代码。
