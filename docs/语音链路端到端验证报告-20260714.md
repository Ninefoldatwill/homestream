# HomeStream VoiceBridge 语音链路端到端验证报告（2026-07-14）

## 一、问题背景
九重 7/14 上午亲测语音交互反馈「还是没有回话」：浏览器显示连接成功、麦克风有声、VAD 检测到「用户说话中」并进入「正在思考」，但**始终没有「正在说话」**。

## 二、根因定位
查看 Worker 日志，核心报错：
```
APITimeoutError: Request timed out.
failed to generate LLM completion after 4 attempts
Agent 状态: thinking -> listening  （没有到 speaking）
```
**根因**：用户说话被 STT 识别后触发 LLM（Ollama qwen2.5:3b），但 Ollama **首次冷启动加载模型约 27s**，超过旧 `_TimeoutLLM` 的 **60s** 默认上限；LiveKit OpenAILLM 内部 4 次 retry 叠加后失败，对话直接回 listening，不出声。

> 注：非语音链路问题，STT / VAD / TTS 均正常。

## 三、修复方案
1. **超时放宽**：`_TimeoutLLM` 默认超时 `60s → 180s`，覆盖冷启动 + 推理余量。
2. **启动预热**：Worker 启动时主动调用一次 Ollama 推理（实测预热后响应 **1.3s**），避免首次说话还要等模型加载。
3. 重启 Worker 生效。

## 四、端到端自动化验证（不依赖人工麦克风）
新增 `test_voice_e2e_auto.py`：
- EdgeTTS 生成中文语音「你好，请简单介绍一下你自己」→ ffmpeg 转 raw PCM(16-bit/48k)；
- 作为「用户说话」经 LiveKit 麦克风轨道发进 `voice-test` 房间；
- 监听 Agent 回的音频轨道，统计出声音频字节。

**验证结果 ✅ 全链路出声成功：**
| 环节 | 结果 |
|------|------|
| Agent 进房 | ✅ `initializing → listening` |
| STT（FunASR 2-pass） | ✅ 准确识别「你好，请简单介绍一下你自己」 |
| LLM（Ollama qwen2.5:3b） | ✅ 进入 thinking，**无超时**（180s 保护生效） |
| TTS（EdgeTTS 兜底） | ✅ 流式完整出声 **10.3s / 984960 bytes** |

Worker 日志佐证：`STT 识别结果: 你好，请简单介绍一下你自己` → `Agent 状态: listening -> thinking` → `using preemptive generation`（边想边出声）。

## 五、关键坑点（脚本层）
LiveKit 自动给 Agent 的 participant `identity` 是 `agent-xxxx`，**不是** `agent_name=homestream-voice`。脚本按 agent_name 过滤会漏掉音频轨道，导致误判「未出声」。已改为收集房间内任意远端音频轨道（房间只有用户 + Agent 两人）。

## 六、结论
语音链路稳定性问题（Ollama 冷启动 LLM 超时）**已彻底修复并自动化验证**。当前 v5.2.5 语音栈：LiveKit(:7880) + FunASR(:10096) + Ollama(:11434) + Worker(homestream-voice) + demo(:8000) 全链路就绪，可正常对话。
