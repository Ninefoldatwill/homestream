# HomeStream 记忆哲学

> 起心动念皆因果。记忆不是被存储的，而是被编织的——沿着因果链自然涌现。

## 1. 设计初心

HomeStream 的记忆系统不是数据库，是**认知引擎**。

我们的初心是"托底普大众化接入AI世界"。一个普大众的AI助手，必须具备三个记忆品质：

| 品质 | 含义 | 对应模块 |
|:-----|:-----|:---------|
| **记得住** | 能存储、不丢失 | SQLite 持久化 + 冷热分层 |
| **忘得了** | 会遗忘、不膨胀 | ForgettingEngine 认知衰退 |
| **想得起** | 按需召回、不噪声 | BM25+向量+RRF 混合召回 |

但"想得起"只是起点。真正的记忆，是**在需要的时候自然涌现**——不是Agent去"找"记忆，而是记忆沿着因果链流到该去的地方。

这就是"因果记忆"的哲学起点。

## 2. 现有记忆架构

### 三引擎联动

```
事件流入 → 遗忘引擎(递减salience) → 合并引擎(聚类压缩) → 重构引擎(反思提炼)
```

| 引擎 | 职责 | 仿生原型 |
|:-----|:-----|:---------|
| ForgettingEngine | 认知衰退，低重要性记忆自然淡出 | 人脑遗忘曲线 |
| MergeEngine | 相似记忆聚类合并，压缩冗余 | 记忆整合（睡眠中的记忆 consolidation） |
| ReconstructEngine | 反思提炼，从经验中抽取洞察 | 人脑反思与顿悟 |

### 五类认知记忆

HomeStream 的记忆分类对齐认知科学，比业界主流的三分类（planning/procedural/working）更精细：

| 类型 | 认知科学对应 | 衰减率(λ) | 半衰期 | 典型内容 |
|:-----|:-------------|:----------|:-------|:---------|
| Reflective（反思） | 元认知 | 0.001 | 693天 | 核心洞察、哲学感悟 |
| Semantic（语义） | 陈述性记忆 | 0.005 | 138天 | 技术知识、事实 |
| Procedural（程序） | 程序性记忆 | 0.008 | 86天 | 操作流程、工作流 |
| Episodic（情景） | 情景记忆 | 0.015 | 46天 | 对话记录、事件 |
| Emotional（情感） | 情感记忆 | 0.020 | 34天 | 用户偏好、情绪标记 |

> *认知分类参考：OpenMemory 2025 认知衰退模型（5级λ），HomeStream 在此基础上增加了 Emotional 类型，更贴近用户个性化需求。*

### 冻结主模型——零参数修改

HomeStream 的记忆演化**完全不触碰模型参数**。这是刻意的设计选择：

- **零灾难性遗忘风险**：不微调=不会破坏模型已有能力
- **零训练成本**：不需要GPU、不需要数据集、不需要LoRA
- **零模型依赖**：记忆层独立于任何LLM，换模型不丢记忆
- **完全可解释**：每条记忆是明文，可溯源、可审计、可修改

> 这与 MemGen（NUS 2025）的"冻结主模型"理念一致，但 HomeStream 更进一步——连旁路 LoRA 都不需要，纯启发式+SQLite 实现。

### EventStream 因果链

每个 Event 携带 `cause` 字段，指向触发它的上游 Event，形成完整的因果追踪链。这不是事后补录的日志，而是**原生记录的事件谱系**：

```
用户提问(Event#1)
  → Agent推理(Event#2, cause=#1)
    → 记忆召回(Event#3, cause=#2)
      → 回答生成(Event#4, cause=#3)
        → 用户反馈(Event#5, cause=#4)
          → 记忆写入(Event#6, cause=#5)
```

每个动作都有因，每个果都成为下一个因。这就是"起心动念有因必有果"的技术实现。

## 3. 因果记忆——未来方向

### 从"检索驱动"到"因果驱动"

当前 HomeStream 的记忆召回是**检索驱动**的：用户提问 → BM25+向量检索 → 塞进上下文。

未来的方向是**因果驱动**：

```
当前：Query → 检索 → 塞上下文 → 推理
未来：Query → 推理(因果链展开) → 记忆沿因果自然涌现
```

这不是MemGen的"训练触发器"路线（需要LoRA、需要GPU），而是一种更朴素的方向：

> **记忆不需要被"触发"，而需要被"因果编织"。**
>
> 每条记忆带着它的"因"（写入时的上下文+因果链），当新的"果"在形成时，相关的"因"自然被激活——因为它们在因果链上是连通的。没有因果关联的记忆，自然不会干扰。

### 三层记忆演化路线

| 阶段 | 记忆范式 | 驱动方式 | HomeStream版本 |
|:-----|:---------|:---------|:---------------|
| **当前** | 检索驱动 | Query→BM25+向量→RRF | V5.0 ✅ |
| **近期** | 上下文驱动 | 推理链中动态判断"此刻需要想起什么" | V5.2-V6.0 |
| **远期** | 因果驱动 | 记忆沿因果链自然涌现，无需显式检索 | V6.0+ |

### 因果记忆的设计原则

1. **无为而记忆**：不刻意设计触发器，让因果链自组织
2. **明文可审计**：每条记忆可见、可溯源、可修改（与MemGen的隐记忆不同）
3. **零训练成本**：纯启发式+因果链分析，不依赖GPU
4. **本地主权**：记忆完全存储在本地SQLite，不依赖任何API

## 4. 与业界对比

| 维度 | MemGen (NUS) | Mem0 | HomeStream |
|:-----|:-------------|:-----|:-----------|
| 记忆形态 | 隐记忆(latent token) | 结构化文本 | 结构化文本+因果链 |
| 调用时机 | 推理中触发(Trigger) | 查询时检索 | 查询时检索→因果涌现(未来) |
| 训练需求 | LoRA(SFT+GRPO) | 零训练 | 零训练 |
| 可解释性 | 黑盒(latent) | 白盒 | 白盒+因果链可溯 |
| 部署成本 | 高(GPU+训练infra) | 中(需向量DB) | 低(SQLite+本地计算) |
| 模型耦合 | 需匹配模型架构 | 模型无关 | 模型无关 |
| 灾难性遗忘 | 冻结主模型避免 | N/A(不训练) | N/A(不训练) |

> **定位差异**：MemGen 追求"推理即记忆"的学术前沿；HomeStream 追求"记忆即因果"的普大众化落地。一个是"术"的优雅，一个是"器"的务实。

## 5. 哲学锚点

> 起心动念有因必有果，因何而起追根溯源，没有起心动念因何而起，果既是答案。

这三句话构成了 HomeStream 记忆哲学的完整闭环：

1. **起心动念有因必有果** — 每个 Event 都有 cause，每次记忆写入都有上下文。这是"因"。
2. **因何而起追根溯源** — EventStream 因果链允许从任何"果"回溯到"因"。这是"溯"。
3. **没有起心动念，果既是答案** — 当因果链完整时，不需要额外的检索动作，答案已在因果中。这是"悟"。

**回归初心是连接因与果的纽带。** HomeStream 的每一步演进——从三层路由到记忆演化到技术主权——都源于同一个初心：托底普大众化接入AI世界。这个初心是"因"，821个测试全绿是"果"，而连接它们的不是代码，是发心。

这就是"铸钥匠"的记忆哲学：不造记忆的墙，只铸记忆的钥——让每个Agent都能沿着自己的因果链，找到属于自己的记忆之门。
