## 二、关键环境调研结论（先调研再动手）

| 项 | 现状 | 影响 |
|---|---|---|
| GPU | ✅ RTX 4050 Laptop GPU，6GB 显存，仅占 2.1GB | CosyVoice2-0.5B(~500MB) 可本地 GPU 推理 |
| Conda | ❌ 未安装（已新装 Miniconda 至 `E:\miniconda3`） | CosyVoice2 必须 Python 3.10 Conda |
| Python 3.10 | ❌ 不存在（将由 Conda 提供） | — |
| Worker 运行环境 | 3.13 venv（`envs/default`） | 无法 import cosyvoice（pynini/Matcha-TTS/kaldifst 原生依赖） |
| 磁盘 | C 盘仅 9.9GB（紧张）；E 盘 878GB | Conda 环境与模型权重放 **E 盘** |

**核心架构决策（调研锁定）：**
> CosyVoice2 依赖 pynini / Matcha-TTS / kaldifst 等大量原生库，官方要求 Python 3.10 + Conda，
> 无法塞进 Worker 当前的 3.13 venv。因此采用**独立 TTS 微服务**架构：
>
> ```
> Worker (3.13 venv)  --HTTP /synthesize-->  CosyVoice2 微服务 (Conda 3.10, :50000)
>                                  <-- audio/wav (24kHz mono)
> ```
> Worker 端 `CosyVoiceTTS` 改为 HTTP 客户端适配器；微服务不可达时自动降级 EdgeTTS。

## 三、已完成的代码改动

1. **`voice/cosyvoice_service.py`（新增）** — 独立微服务（运行于 Conda 3.10）。
   - `GET /health`、`GET /voices`、`POST /synthesize`（JSON→WAV）。
   - 懒加载 CosyVoice2 **`CosyVoice2(model_dir, fp16=False)`**（**注意：没有 `use_flow_cache` 参数**，那是旧版 CosyVoice 的，传了会 TypeError）。
   - **音色策略（自动分支）**：`list_available_spks()` 返回非空且指定 voice 在内 → `inference_sft(tts_text=, spk_id=, ...)`；否则（CosyVoice2-0.5B 官方发布**不含 `spk2info.pt`**）→ 自动降级 `inference_zero_shot(tts_text=, prompt_text="希望你以后能够做的比我还好呦。", prompt_wav=CosyVoice/asset/zero_shot_prompt.wav)`，用官方自带提示音做音色克隆，保证出声。
   - **多句拼接修复**：`text_normalize(split=True)` 会把长文本拆多句、每句 yield 一个 chunk，循环必须 `np.concatenate` 所有 `tts_speech`，否则多句回复只剩最后一句音频。
   - `wave` 标准库编码 WAV（零额外 web 依赖）；路径自动发现（在 sys.path 注入 `CosyVoice/` 与 `CosyVoice/third_party/Matcha-TTS/`，无需 `pip install`）。
2. **`voice/tts_adapter.py`（重写 CosyVoice 部分）** — 移除进程内 `cosyvoice` 导入（在 3.13 下必失败），改为：
   - `CosyVoiceClient`：HTTP 探活 `/health` + `/synthesize` 调用。
   - `CosyVoiceTTS`（LiveKit 1.6.5 `ChunkedStream`）：拿到 WAV → 解码为 float32 → 按 ~100ms 分片 `output_emitter.push`，由 AgentSession 的 StreamAdapter 桥接流式。
   - `create_tts()`：先探活微服务，**可达则优先 CosyVoice2**，否则降级 EdgeTTS（不影响现有链路）。
3. **`voice/start_cosyvoice_service.bat`（新增）** — 一键启动微服务（激活 `cosyvoice` 环境并指定模型目录）。

### 构建坑（本机已踩，已解决 / 收尾中）

- **坑 A — Matcha-TTS 需 C++ 编译器（本机无 MSVC/gcc）**：`pip install -e Matcha-TTS` 要编译 `monotonic_align.core` Cython 扩展，但本机没有任何 C/C++ 编译器 → 编译失败。
  **解法**：`monotonic_align.core.maximum_path` **只在训练 `forward()` 里被调用，推理从不调用**。已用纯 Python 桩 `third_party/Matcha-TTS/matcha/utils/monotonic_align/core.py`（`maximum_path_c`）替代该扩展；并在服务里把 `Matcha-TTS/` 目录直接加入 `sys.path`（不 `pip install`），`import matcha` 即可解析。
- **坑 B — `openai-whisper` 构建隔离拉到最新 setuptools（无 `pkg_resources`）**：`cosyvoice` 顶层 `import whisper`（frontend.py:20），必需 `openai-whisper`；但其 legacy sdist 构建需 `pkg_resources`，而 pip 默认 **build isolation** 会拉最新 setuptools(≥81) 已移除 `pkg_resources` → 构建失败，且 pip 会**整体回滚**导致其余依赖也没装上。
  **解法**：① 本环境 `pip install "setuptools==80.9.0"`（含 `pkg_resources`）；② `pip install --no-build-isolation openai-whisper==20231117`（复用本环境 setuptools）；③ 其余依赖正常装。
- **坑 C — Windows `PYTHONPATH` 分隔符是 `;` 不是 `:`**：校验命令若用 `:` 分隔，Python 会把整串当成一个不存在的路径，`import matcha` 失败。服务代码用 `sys.path.insert()`（Python 层）不受影响；**批处理/脚本里务必用 `;`**。
- **坑 D — 镜像未配置导致 PyPI 极慢/超时**：本环境 `pip` 默认走 pypi.org（国内慢）。已在 `E:\conda_envs\cosyvoice\pip.conf` 写死 TUNA 镜像。

## 五、风险与对策（GFW / Windows）

- **GitHub 克隆失败** → 走国内镜像链（ghfast.top→gitclone.com→直连）。
- **pip 源慢/阻断** → 已在 conda 环境 `pip.conf` 写死 TUNA 镜像；装 `openai-whisper` 须 `--no-build-isolation` + `setuptools<81`。
- **模型下载慢** → modelscope 国内源 + git-lfs；必要时手工下载 `CosyVoice2-0.5B` 压缩包。
- **CUDA 架构不匹配（RTX 4050 = sm89）** → 已选 `pytorch-cuda=12.1`（支持 sm89）。
- **sox 缺失** → 不影响 `inference_*`（`inference_sft`/`inference_zero_shot` 前端用 onnxruntime + whisper，不依赖 sox），暂不装。
- **无 C/C++ 编译器** → Matcha-TTS 用纯 Python 桩替代 Cython 扩展（见坑 A），不编译。
